## 📌 О проекте

Разработанная система обеспечивает **централизованный мониторинг** оборудования сети доступа (DSLAM, ONU/OLT, Ethernet-коммутаторы, маршрутизаторы) с использованием протокола **SNMP v2c**. Решение автоматизирует сбор, обработку и визуализацию данных о загрузке CPU, использовании оперативной памяти, дисковой ёмкости и сетевых интерфейсах.

Система ориентирована на малые и средние организации связи, где важны простота внедрения, минимальные затраты на сопровождение и наглядность.

### 🎯 Цель работы

Создание программного обеспечения для автоматизированного мониторинга и учёта ресурсов телекоммуникационного оборудования сети доступа с использованием SNMP.

### 📋 Задачи

- Анализ предметной области и существующих аналогов (Nagios, Zabbix, PRTG, SolarWinds, OpManager)
- Проектирование архитектуры (MVC, UML-диаграммы)
- Разработка модуля сбора данных по SNMP
- Создание базы данных (MariaDB) для хранения информации
- Реализация веб-интерфейса (SPA) для визуализации и управления
- Тестирование и сравнительная оценка характеристик

---

## 🏗 Архитектура

- **Паттерн:** MVC (Model-View-Controller)
- **Клиентская платформа:** веб-приложение (SPA)
- **Сервер:** Apache + PHP
- **БД:** MariaDB
- **Протокол мониторинга:** SNMP v2c (MIB: IF-MIB, HOST-RESOURCES-MIB, SNMPv2-MIB)

---

## 🚀 Основной функционал

| Модуль | Описание |
|--------|----------|
| **Обнаружение устройств** | Автоматическое сканирование локальной сети (ARP) |
| **SNMP-опрос** | Сбор данных по OID (CPU, RAM, диски, интерфейсы) |
| **Хранение** | Нормализованная БД с внешними ключами |
| **Веб-интерфейс** | Таблицы с вложенными секциями (интерфейсы, ресурсы) |
| **Уведомления** | Логирование успешных/ошибочных SNMP-запросов |
| **Автообновление** | Асинхронная загрузка данных по таймеру (10–60 сек) |

### Собираемые метрики

**Ресурсы устройства:**
- Загрузка CPU (`hrProcessorLoad`)
- Использование ОЗУ (используемая / общая)
- Диски (имя, используется / всего)

**Сетевые интерфейсы:**
- In/Out Unicast пакеты
- In/Out Multicast пакеты

## 🖥 Интерфейс системы

### Главная страница
- Кнопка ручного запуска SNMP-опроса
- Настройка интервала автообновления
- Блок уведомлений (ошибки/успех)
- Таблица устройств с кнопками раскрытия:

### Детализация по каждому устройству
- **Интерфейсы:** таблица с in/out пакетами (unicast/multicast)
- **Ресурсы:** CPU, диски (имя, использование/объём), RAM

> 📌 Все данные обновляются асинхронно (fetch API) без перезагрузки страницы.

---

## 🗄 Структура базы данных (MariaDB)

| Таблица | Содержание |
|---------|------------|
| `devices` | ID, hostname, IP, маска, логин, пароль |
| `interface_traffic` | device_id, интерфейс, in/out unicast/multicast |
| `device_resources` | device_id, CPU, memory_usage, memory_total |
| `disks` | device_id, disk_name, disk_usage, disk_total |
| `log_messages` | статус, сообщение (успех/ошибка SNMP) |
